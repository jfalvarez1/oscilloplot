// Introduction //

Oscilloplot C++ is a high-performance, cross-platform application for generating XY audio signals that can be displayed on a real analog oscilloscope. This complete rewrite in C++ delivers dramatically improved performance with a physically-accurate CRT phosphor simulation inspired by the legendary Tektronix 465B.

Key Features

Realistic Phosphor Display

Physics-based CRT simulation with velocity-dependent brightness, exponential decay, and P31/P1/P7/Amber phosphor presets.

120+ FPS Performance

Native C++ with OpenGL acceleration delivers buttery-smooth real-time visualization at minimal CPU usage.

3D Shape Generator

Project rotating 3D shapes (cube, torus, icosahedron, knots) to 2D for mesmerizing wireframe animations.

Audio Output

Generate stereo audio signals (Left = X, Right = Y) for real oscilloscope display with ultra-low latency.

Multiple Generators

Sum of Harmonics with frequency sweep, Archimedean Spirals, test patterns, and freehand drawing.

15+ Effects

Rotation, fade, shrink, wavy, tremolo, ring mod, echo, kaleidoscope, distortion, and more.

// Phosphor Display Simulation //

The C++ edition features a physically-accurate CRT phosphor simulation based on the Tektronix 465B oscilloscope.

How Real CRT Oscilloscopes Work

In a real cathode ray tube oscilloscope, an electron beam is deflected by X and Y voltages to trace patterns on a phosphor-coated screen. The physics create characteristic visual effects:

Velocity-Dependent Brightness

Brightness ∝ 1 / Velocity

When the beam moves slowly, it deposits more electrons per unit length, causing the phosphor to glow brighter. Fast-moving traces appear dimmer. This creates natural visual emphasis on corners, endpoints, and slow sections.

Phosphor Persistence (Decay)

I(t) = I0 × e-t/τ

After the beam passes, the phosphor continues to glow with exponentially decaying intensity. The time constant τ varies by phosphor type. P31 (green) has ~50ms visible persistence, creating the classic "trailing" look.

Glow/Bloom Effect

Light scatters within the phosphor layer, creating a soft glow around the beam trace. This scatter follows an exponential falloff (not Gaussian), more concentrated near the beam center.

Phosphor Presets

PresetColorCharacteristicsUse Case
P31 Yellowish-Green ~50ms decay, medium persistence Classic Tektronix look (default)
P1 Pure Green ~20ms decay, short persistence Fast signals, modern look
P7 Blue-White ~300ms decay, long persistence Single-shot capture, slow signals
Amber Orange ~35ms decay, warm tone Vintage computing aesthetic

How much beam velocity affects brightness. 0 = uniform, 1 = full velocity dependence.

Range: 0.5 - 3.0 | Default: 1.2

Core beam trace thickness in pixels.

Beam Focus
Range: 0.5 - 1.0 | Default: 0.8

Beam sharpness. 1.0 = sharp focused beam, lower = softer/defocused.

Range: 0.0 - 1.0 | Default: 0.5

Strength of the phosphor bloom/glow effect.

Glow Radius
Range: 1 - 8 pixels | Default: 3

Spread of the bloom effect around the beam.

Trail Samples
Range: 1024 - 16384 | Default: 8192

Number of samples displayed at once. More samples = longer visible trail.

// Sum of Harmonics Guide //

Create Lissajous figures by combining sine and cosine terms. The C++ version adds frequency sweep for morphing animations.

Mathematical Basis

X(t) = Σ Ai · sin(fi · t + φi)
Y(t) = Σ Bj · cos(gj · t + ψj)

Classic Lissajous Ratios

X:Y RatioShape
1:1Circle, Ellipse, or Line
1:2Figure-8 (infinity)
2:3Pretzel / Bow-tie
3:4Complex interlocking loops
5:45-petal flower

New: Frequency Sweep

Enable Freq Sweep on any term to animate its frequency from a start to end value. This creates dramatic morphing effects where patterns transform from one shape to another.

Sweep Examples

StartEndEffect
1.05.0Circle → 5-petal flower
2.03.0Figure-8 → Pretzel

// Connecting to an Oscilloscope //

Step-by-Step Setup

  1. Set oscilloscope to XY mode
    Consult your manual for the specific setting (also called X-Y, dual channel, or Lissajous mode).
  2. Connect audio output
    • Left channel → X input (Channel 1)
    • Right channel → Y input (Channel 2)
    Use a stereo 3.5mm to dual BNC/RCA adapter cable.
  3. Adjust oscilloscope settings
    • Set time/div to external or X-Y mode
    • Set coupling to AC or DC
    • Adjust gain on both channels equally
  4. Start playback
    Press Space or click Play in Oscilloplot.
  5. Fine-tune display
    Adjust oscilloscope intensity, focus, and position.

Tektronix 465B Specific Settings

// Troubleshooting //

No Audio Output

  • Check system audio output device is set correctly
  • Verify volume is not muted
  • Try reducing the playback multiplier if sample rate is too high for your sound card

Display Looks Wrong

  • Ensure you've generated a pattern first
  • Check Display Settings - try resetting to P31 preset
  • Increase Trail Samples if pattern appears incomplete

Pattern Looks Wrong on Real Oscilloscope

  • Check X/Y channel connections aren't swapped
  • Ensure both channels have equal gain settings
  • Verify oscilloscope is in XY mode, not time-base mode
  • Adjust audio output volume

Application Crashes on Startup

  • Ensure your GPU drivers are up to date (OpenGL 3.3 required)
  • Check that all DLL dependencies are present (Windows)
  • Try running from command line to see error messages

Low Frame Rate

  • Reduce Trail Samples in Display Settings
  • Disable Glow effect (set intensity to 0)
  • Close other GPU-intensive applications
